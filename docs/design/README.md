## 🎯 設計原則

MarketVue 的設計遵循以下核心原則：

### 1. 專業且富有個性
金融工具不需要犧牲個性來維持專業形象。透過 thoughtful 的動效設計、氛圍背景、和獨特的字體排印，MarketVue 在保持可信度的同時脫穎而出。

### 2. 雙主題並存
- **Classic**: 現代、專業、清晰 - 適合傳統金融用戶
- **Warm Minimal**: 溫暖、精緻、優雅 - 適合追求美感的用戶

兩種主題各有特色，滿足不同用戶偏好。

### 3. 數據為先
無論視覺設計如何華麗，清晰呈現數據永遠是第一優先。所有設計決策都需要支持資訊傳遞。

### 4. 漸進式增強
從功能性基礎開始，逐步加入視覺精緻度。確保在動畫關閉或低性能設備上仍能正常使用。

## 🎨 當前設計狀態

### Classic 主題
- **色彩**: 藍色漸變 + 灰階
- **字體**: 系統字體
- **風格**: 現代、清晰、專業
- **狀態**: ✅ 穩定，可用於生產環境

### Warm Minimal 主題
- **色彩**: 米色中性色調 + 赤陶色強調色
- **字體**: Playfair Display (襯線) + Inter (無襯線)
- **風格**: 溫暖、優雅、精緻
- **狀態**: ✅ v1.6.0 新增，可用於生產環境

## 🚀 待優化領域

根據最新設計審視報告，以下領域有優化空間：

### 🔴 高優先級
1. **字體排印**: 替換 Inter 為更有特色的字體
2. **動效設計**: 加入頁面載入動畫、數字動畫、懸停微互動
3. **背景質感**: 從純色升級為漸變網格、噪點紋理

### 🟡 中優先級
4. **空間構圖**: 打破僵硬的網格，加入非對稱元素
5. **色彩優化**: 擴展 Warm 主題的強調色板
6. **互動狀態**: 增強按鈕、載入、錯誤狀態的視覺回饋

### 🟢 低優先級
7. **深色模式優化**: 加入環境光效果
8. **自訂游標**: 添加情境化游標
9. **捲動觸發**: 實作捲動顯示動畫

詳細說明與實作建議請參考 [設計審視報告](DESIGN_REVIEW_AND_RECOMMENDATIONS.md)。

## 💡 貢獻指南

### 提出設計建議

1. 在 GitHub Issues 中開一個新 issue
2. 使用 `design` 標籤
3. 清楚描述：
   - 當前問題或改進機會
   - 建議的解決方案
   - 預期影響（用戶體驗、性能、可維護性）
   - 參考資源或範例

### 提交設計變更

1. Fork 專案並創建新分支
2. 實作設計變更
3. 確保符合可訪問性標準（WCAG AA）
4. 測試不同瀏覽器和裝置
5. 更新相關文件
6. 提交 Pull Request
