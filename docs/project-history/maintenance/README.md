# Maintenance Work Logs

這個目錄存放專案維護相關的工作紀錄，包括流程改進、規範建立、審查報告等。

## 目錄用途

- **流程改進**：建立或改進開發流程的紀錄
- **規範建立**：創建或更新開發規範的紀錄
- **審查報告**：程式碼審查、版本號審查等報告
- **維護紀錄**：重構、升級、清理等維護工作

## 與其他目錄的區別

- **`phases/`**：功能開發相關的工作紀錄（Phase 1-3）
- **`optimizations/`**：效能優化相關的工作紀錄
- **`deployments/`**：部署驗證相關的工作紀錄
- **`maintenance/`**（本目錄）：流程改進和維護相關的工作紀錄

## 命名規範

```
work-log-{主題}-{YYYY-MM-DD}.md
```

例如：
- `work-log-versioning-audit-2025-12-09.md` - 版本號審查
- `work-log-testing-framework-upgrade-2025-12-15.md` - 測試框架升級
- `work-log-code-quality-review-2025-12-20.md` - 程式碼品質審查
